以下の手順でPRを作成してください

## 手順

### Step 1: ブランチ状態の確認

現在のブランチ名、未コミットファイル、直近のコミット履歴を確認し、PR作成の準備が整っているかを判断する。

**必要なコマンド**:

```bash
git status
git log --oneline -5
git branch --show-current
```

### Step 2: 変更差分の把握

mainブランチからの差分統計を確認する。詳細なdiffは不要。

**必要なコマンド**:

```bash
git diff main...HEAD --stat
```

### Step 3: PRテンプレート取得と本文作成

`Read`ツールでPRテンプレート（`.github/PULL_REQUEST_TEMPLATE.md`）を取得し、変更内容に基づいて各項目を埋める。

**本文作成の要点**:

- **Overview**: 変更の目的と価値を簡潔に説明
- **Changes**: 変更の意味的なまとまりごとに階層化して記述

**注意**: コミットメッセージやgit diffの内容を活用して、効率的に本文を構成する。

### Step 4: PR作成

`gh pr create`コマンドを使用してPRを作成する。

**コマンド形式**:

```bash
gh pr create \
  --title "日本語の分かりやすいタイトル" \
  --body "$(cat <<'EOF'
[PRテンプレートに基づいた本文]
EOF
)" \
  --base main \
  --head <現在のブランチ名>
```

**パラメータ説明**:

- `--title`: ブランチ名から適切なタイトルを生成（日本語、簡潔で分かりやすく）
- `--body`: heredocを使用してPR本文を記述（テンプレート準拠）
- `--base`: マージ先ブランチ（通常は`main`）
- `--head`: 現在のフィーチャーブランチ名
- `--draft`: オプションで指定された場合に追加（ドラフトPR作成）

**成功時**: PR URLが出力される

## Options

- `--draft`: PRをドラフトとして作成する

## コンテキスト効率化のポイント

1. **詳細diffは読まない**: `git diff --stat`で統計のみ確認
2. **コミットメッセージを活用**: `git log`の情報を本文に反映
3. **テンプレートは1回のみ読む**: Readツールで取得し、メモリ上で編集
4. **heredocで直接記述**: 一時ファイルを作らず、コマンド内で本文完成
