# Hub画面: レイアウト仕様

[<< Hub画面設計トップに戻る](../0003.hub.md)

## 1. レイアウトの基本方針

デバイスの特性に応じて、最適な情報密度と操作性を提供するために、PCとモバイルで異なるレイアウト戦略を採用します。

- **PC (ワイドスクリーン)**:

  - **3カラムレイアウト**を採用し、情報の「**全体像**」「**詳細**」「**操作**」を一覧表示します。これにより、ユーザーは視線移動のみで、複数の情報を比較・検討しながら効率的に探求作業を行えます。

- **モバイル (省スペース)**:
  - **タブ切り替えレイアウト**を採用します。画面上部に「Hub（Canvas）」を固定表示し、下部の「タブバー」で「操作（Controller）」と「情報（Information）」を切り替えることで、限られた画面領域内で効率的な作業フローを実現します。

## 2. PCレイアウト

### 2.1. 画面構成

| エリア | コンポーネント名          | 役割                                                         |
| :----- | :------------------------ | :----------------------------------------------------------- |
| **A**  | グローバルヘッダー        | アプリケーション全体のナビゲーションを提供します。           |
| **B**  | メイン表示エリア (Canvas) | 音楽理論の視覚的表現（Hub）を描画する中心領域です。          |
| **C**  | コントロールパネル        | Canvasに表示する内容（ビュー、キー、レイヤー）を制御します。 |
| **D**  | インフォメーションパネル  | 選択要素や現在のキーに関する詳細情報を表示します。           |

## 3. モバイルレイアウト

### 3.1. 画面構成

| エリア | コンポーネント名          | 役割                                                                               |
| :----- | :------------------------ | :--------------------------------------------------------------------------------- |
| **A**  | グローバルヘッダー        | アプリケーション全体のナビゲーションを提供します。                                 |
| **B**  | メイン表示エリア (Canvas) | 画面上部に固定表示され、音楽理論の視覚的表現（Hub）を描画します。                  |
| **C**  | コンテンツエリア          | Canvasの下に配置され、選択されたタブの内容（Controller/Information）を表示します。 |
| **D**  | タブバー                  | 画面下部に固定され、コンテンツエリアの表示内容を切り替えます。                     |

### 3.2. 構成イメージ

```
+------------------------------------------+
| A. グローバルヘッダー                    |
+------------------------------------------+
|                                          |
| B. メイン表示エリア (Canvas)             |
|   [現在のキー: C Major]                  |
|    五度圏 / クロマチックサークル         |
|   (上部固定・スクロールしない)           |
|                                          |
+------------------------------------------+
|                                          |
| C. コンテンツエリア (スクロール可能)     |
|                                          |
|  [Controller タブ選択時]                 |
|   - C-1: ViewController                  |
|   - C-2: KeyController                   |
|   - C-3: LayerController                 |
|                                          |
|  [Information タブ選択時]                |
|   - D-1: 選択要素情報                    |
|   - D-2: 現在のキー情報                  |
|                                          |
+------------------------------------------+
| D. タブバー (下部固定)                   |
| [ Settings (Controller) ] [ Info (Information) ] |
+------------------------------------------+
```

### 3.3. モバイル特有のUI/UX仕様

#### タブバー (`D`)

- **役割**: 限られた画面領域を有効活用するため、操作系（Controller）と情報系（Information）を明確に分離し、ワンタップで切り替えます。
- **配置**: 画面最下部に固定（Sticky Footer）。
- **項目**:
  - **Controller**: 設定・操作パネルを表示。
  - **Information**: 詳細情報パネルを表示。

#### Canvas (`B`)

- **配置**: ヘッダー直下に固定表示。
- **挙動**: コンテンツエリアをスクロールしても、Canvasは常に表示され続けます。これにより、ユーザーは常に「地図（Hub）」を見ながら、情報の確認や操作を行えます。

#### コンテンツエリア (`C`)

- **スクロール**: Canvasとタブバーの間の領域で独立してスクロールします。
